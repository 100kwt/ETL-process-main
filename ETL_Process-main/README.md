# Учебный проект: ETL-процесс и витрина мошеннических операций

## Описание проекта

Проект реализует ETL-процесс для загрузки исходных данных в хранилище PostgreSQL, обработки транзакций и формирования витрины мошеннических операций.

После выполнения всех шагов создаются необходимые таблицы, загружаются данные, обрабатываются исходные файлы из директории `data`, формируется витрина данных, а также создается аналитический отчет `fraud_analysis.txt`.

## Порядок запуска проекта

### 1. Настройка подключения к PostgreSQL

Перед запуском проекта необходимо настроить подключение к базе данных PostgreSQL.

Необходимо указать параметры подключения.

### 2. Создание необходимых таблиц

После настройки подключения необходимо выполнить SQL-скрипт `create_tables.sql`, который находится в директории `sql_scripts`.

Скрипт создает необходимые таблицы в базе данных PostgreSQL.

### 3. Наполнение таблиц данными

После создания таблиц необходимо выполнить SQL-скрипт `ddl_dml.sql`, который также находится в директории `sql_scripts`.

Скрипт наполняет таблицы необходимыми данными.

### 4. Подготовка исходных данных

Исходные данные должны находиться в директории `data`.

Основной скрипт проекта работает с файлами, расположенными в этой директории.

### 5. Запуск основного скрипта

После создания и наполнения таблиц необходимо запустить основной файл проекта main.py

Файл `main.py` обрабатывает исходные данные из директории `data`, загружает их в базу данных и формирует необходимые результаты.

### 6. Результат выполнения

После успешного выполнения `main.py` будут получены:

- необходимые данные в базе данных PostgreSQL;
- итоговая витрина данных;
- все необходимые обработанные данные;
- аналитический отчет в файле `fraud_analysis.txt`.

Файл с аналитическим отчетом создается после завершения работы основного скрипта/
